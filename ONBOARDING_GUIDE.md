## 📋 概述

新手引导系统采用**渐进式引导**策略，在用户首次使用某个功能时显示简短提示，避免启动时的全屏教程打断用户体验。

**核心原则：**
- ✅ 不打断用户流程
- ✅ 只在必要时出现
- ✅ 每个引导只显示一次
- ✅ 提供明确的操作指引

## 🔧 技术实现

### OnboardingManager

**文件：** `OnboardingManager.kt`

**核心功能：**
1. 追踪用户是否已看过某个引导
2. 确保每个引导只显示一次
3. 使用Snackbar显示提示
4. 支持自定义操作按钮

**主要方法：**

```kotlin
// 显示引导（只显示一次）
OnboardingManager.showGuideOnce(
    context = this,
    view = binding.root,
    guideType = OnboardingManager.GuideType.JOYSTICK_USAGE,
    message = "提示信息",
    actionText = "知道了"
)

// 检查是否已显示过
val hasShown = OnboardingManager.hasShownGuide(context, guideType)

// 重置引导状态（用于测试）
OnboardingManager.resetAllGuides(context)
OnboardingManager.resetGuide(context, guideType)
```

## 🔄 工作流程

```
用户首次触发功能
  ↓
检查是否已显示过引导
  ↓
├─ 已显示 → 直接执行功能
└─ 未显示 → 显示引导提示
     ↓
  用户看到提示
     ↓
  点击"知道了"或自动消失
     ↓
  标记为已显示
     ↓
  下次不再显示
```

## 📝 最佳实践

### 1. 引导文案设计

**✅ 好的文案：**
- "💡 提示：点击FAB按钮后，可以使用摇杆控制移动方向"
- "💡 提示：输入地点名称或地址，点击搜索结果即可定位"

**❌ 不好的文案：**
- "这是摇杆"
- "搜索功能"
- "使用说明"

**原则：**
- 以"💡 提示："开头，统一风格
- 明确指出功能和用途
- 简洁易懂，不超过一行

### 2. 触发时机选择

**✅ 好的时机：**
- 用户主动使用功能时
- 功能上下文清晰时
- 用户有心理准备时

**❌ 不好的时机：**
- App刚启动就显示所有引导
- 与当前操作无关时
- 连续弹出多个引导

### 3. 引导数量控制

**建议：**
- 核心功能：3-5个引导
- 次要功能：可选
- 高级功能：不需要

**当前设置：**
- 已启用3个核心引导
- 预留3个扩展引导

## ⚠️ 注意事项

### 1. 不要过度引导

- ❌ 每个功能都显示引导
- ❌ 同一个引导多次显示
- ❌ 引导文案过长

- ✅ 只引导核心功能
- ✅ 每个引导只显示一次
- ✅ 文案简洁明了

### 2. 尊重用户选择

- 用户可以点击"知道了"快速关闭
- 引导自动消失，不强制阅读
- 允许用户跳过引导

### 3. 及时更新引导

- 功能变更时更新引导文案
- 新功能添加时考虑是否需要引导
- 定期评估引导的有效性
