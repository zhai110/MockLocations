## 闪退修复

### 根因分析
点击"开始模拟"闪退，核心原因：
1. **MapView.onCreate(null) 传 null 崩溃** — 高德 MapView.onCreate() 必须传 Bundle
2. **init() 一次性初始化3个视图太重** — 地图+历史在 Service 启动时同步初始化，任何异常导致后续 NPE
3. **MapView 在 Service context 中创建崩溃** — 高德 MapView 需要 Activity 级别带主题的 Context

### 修复内容
1. **延迟初始化**: init() 只初始化摇杆，地图/历史在用户切换时才初始化 (ensureMapInited/ensureHistoryInited)
2. **ContextThemeWrapper**: MapView 用 `ContextThemeWrapper(service, R.style.Theme_VirtualLocation)` 包裹
3. **onCreate(Bundle())**: MapView.onCreate() 传空 Bundle 而非 null
4. **try-catch 保护**: LocationService.onCreate() 中 init/show 加 try-catch，onStartCommand 中 setCurrentPosition 加安全保护

## 悬浮窗不显示修复

### 根因
JoystickView 构造函数中 `loadLockIcon()` 在 `init{}` 块调用 `Bitmap.createScaledBitmap(bitmap, 0, 0, true)` — `innerCircleRadius` 在 `onMeasure()` 之前为 0，导致 `IllegalArgumentException`。

异常被 try-catch 吞掉 → `joystickLayout` 为 null → `addViewSafe(null)` 跳过 → 悬浮窗不显示但也不闪退。

### 修复
`loadLockIcon()` 中增加 `size <= 0` 保护，跳过缩放。`onMeasure()` 后会重新调用 `loadLockIcon()` 正常加载图标。

### 教训
- 自定义 View 的 `init{}` 块不能依赖尺寸（`onMeasure` 之后才有值）
- 异常被 try-catch 吞掉 = 静默失败，"不闪退"不等于"没问题"
- `Bitmap.createScaledBitmap` 要求宽高 > 0

## 悬浮窗拖动修复

- **问题**: 原来的 `setupDrag` 用 `setOnTouchListener` + `return false`，但 ImageButton 消费了触摸事件，导致容器 `onTouch` 收不到回调，按住按钮区域无法拖动

  - `onTouchEvent`: 执行窗口位置更新
  - 短按（未超过 touchSlop）放行给子 View 处理点击
- 拖动逻辑抽取到 `DragHelper` 辅助类，两个 Layout 共用
- 三个容器全部改为 Drag 版本：摇杆用 `DragLinearLayout`，地图/历史用 `DragFrameLayout`

## 悬浮窗暗黑模式实时切换修复

- **问题**: App 跟随系统暗黑模式可实时切换，但悬浮窗需要重启才能切换
- **根因**: Service 的 Context 不会随系统暗黑模式自动更新 Resources，
  `ContextCompat.getColor(service, R.color.xxx)` 始终读取旧主题颜色
- **修复**:
  1. 新增 `themedContext` 类属性：`ContextThemeWrapper(service, R.style.Theme_VirtualLocation)`
     — 继承 service 的 configuration（含最新 uiMode），叠加主题后自动选择 values/values-night
  2. 所有 `ContextCompat.getColor(service,` → `ContextCompat.getColor(themedContext,`（21 处）
  3. 关键 View 创建改用 `themedContext`：JoystickView、ButtonView、EditText、MapView
  4. `syncMapWithSystemTheme()` 中主题变化时：重建 `themedContext`，先 removeView 再置空缓存再 show()
  5. 修复 `show()` 中 joystickLayout 置 null 后无 ensure 保护：添加 `ensureJoystickInited()`
  6. 删除 initMapView 中局部变量 `themedContext`（遮蔽类属性）

### 悬浮窗暗黑模式实时切换修复（第二轮）
- **根因**: 之前的方案用 `VirtualLocationApp` 注册 `ACTION_CONFIGURATION_CHANGED` 广播 → 发 `ACTION_THEME_CHANGED` → Service 收到后调 `syncMapWithSystemTheme()`。但 `ACTION_CONFIGURATION_CHANGED` 是系统受保护广播，动态注册的 BroadcastReceiver **收不到**
- **修复**: LocationService 直接重写 `onConfigurationChanged()` 回调，系统配置变化时自动调用
- **清理**:
  - VirtualLocationApp: 删除 `themeChangeReceiver`、删除 `ACTION_THEME_CHANGED` 常量、删除相关 registerReceiver/unregisterReceiver 和无用 import
  - LocationService: 删除 `themeReceiver` 成员变量、注册/反注册代码，改用 `onConfigurationChanged`
  - MainActivity: 删除 `themeChangeReceiver`、删除注册/反注册代码、删除无用 import（BroadcastReceiver/IntentFilter/VirtualLocationApp）

### 悬浮窗地图夜景模式同步
- **问题**: 切换暗黑模式时悬浮窗UI颜色正确切换了，但地图仍显示白天模式
- **根因**: `initMapView()` 硬编码 `aMap?.mapType = AMap.MAP_TYPE_NORMAL`

## 设置值传递给服务 — 全面修复

### 问题分析
- 海拔：运行中修改不生效，只在 `onCreate()`/`onStartCommand()` 读取
- 速度：设置页修改后需重新点击按钮才生效，无变更通知
- 随机偏移：`move()` 读SP但 `performAutoMoveStep()` 不读
- 速度模式：重启后丢失，硬编码为 "walk"

### 修复方案
1. **SP 监听器**：`LocationService` 注册 `OnSharedPreferenceChangeListener`
   - `altitude` 变化 → 即时更新 `altitude` 变量
   - `walk_speed`/`run_speed`/`bike_speed` 变化 → 重新应用当前模式
2. **`applySpeedMode()`**：从 SP 读取速度值并更新 `currentSpeed`
3. **`setSpeedMode()`**：切换模式 + 持久化到 SP（`speed_mode` key）
4. **`performAutoMoveStep()`**：加入 `random_offset` 随机偏移逻辑
5. **初始化恢复**：读取 `speed_mode` 恢复上次模式，高亮对应按钮
6. **删除** `FloatingWindowManager.setSpeed()`，改为调用 `service.setSpeedMode()`
